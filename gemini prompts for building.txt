Category A — Patient & MDT identifiers
(patient ID, MDT date, hospital number, DOB, consultant)

Category B — MRI Staging (HIGHEST PRIORITY — Dr. Wale will test these)
Must include separately: mrT stage, mrN stage, mrCRM status (threatened if ≤1mm), 
mrEMVI status, mesorectal fascia involvement, tumour height from anal verge,

Category C — CT Staging

Category D — Pathology & Biomarkers

KRAS/NRAS/BRAF if present.

Category E — MDT Decision
Primary treatment intent (curative vs palliative), treatment modality 
(surgery first / short-course RT / long-course chemoRT / chemotherapy), 
planned surgical procedure (e.g. TME, Hartmann's).

Category F — Safety Flags
Human verification required (boolean), list of fields that triggered the flag, 
overall extraction confidence score.

For each field, specify: field name, data type, accepted values or format, and an 
example of what the source text might look like in the Word document.

Prompt 3 — Stage 2 LLM prompt engineering
Write the actual LLM prompt that will be used in Stage 2 of our pipeline. This prompt 
will be sent to the Gemini API along with the raw segmented text from Stage 1.

Requirements for the prompt:
- It must instruct the model to return ONLY valid JSON, no preamble, no markdown 
  code fences, nothing else
- Every field must follow this exact structure:
  { "value": "...", "source_text": "exact sentence from input", "confidence": "high|low|not_found" }
- It must include explicit instructions on clinical notation — mrT staging must use 
  the mrT prefix, CRM status must note the ≤1mm threshold, EMVI must be recorded as 
  positive/negative/indeterminate
- It must explicitly instruct the model NOT to infer or guess — if a field is not 
  present in the source text, confidence must be "not_found" and value must be null
- It must handle the baseline vs restaging MRI distinction: if the text mentions 
  "post-treatment" or "restaging" MRI, is_restaging_MRI must be set to true
- Include a worked example in the prompt (few-shot): show one example input paragraph 
  and the correct JSON output for it

Also write the Python wrapper function that calls the Gemini API with this prompt, 
handles rate limiting, and validates that the response is parseable JSON before 
returning it. Include try/except with a fallback that marks all fields as 
confidence: "not_found" if the API call fails.
